# 유체 설계 문서

## 1. 개요

유체(FLUX: Fluide Langue Universelle d'Exécution)의 한국어 우선 설계 문서.
본 문서에서는 한국어 문법이 어떻게 프로그래밍 언어의 아키텍처를 근본적으로 형성하는지 설명합니다.

### 핵심 전제

> 한국어는 '프로그래밍 언어로 설계된' 것이 아니다. 하지만 한국어의 문법 구조가
> 컴퓨터 과학의 핵심 개념과 놀라운 정도로 부합한다.
> 이 부합을 의도적으로 설계 원칙으로 삼는다.

## 2. 경어 시스템을 RBAC으로

### 2.1 한국어 경어 체계

한국어에는 문법적으로 규정된 높임말 체계가 있다. 이는 단순한 '예의'가 아니라
문법 규칙이다. 동사 어미의 형태가 화자와 청자의 관계를 결정한다.

| 수준 | 체계 | 동사 어미 예시 | 사용 상황 |
|------|------|---------------|-----------|
| 1 | 해라체 | -다, -라 | 자기에게, 내부 문서 |
| 2 | 해체 | -해, -거라 | 친한 친구, 동료 |
| 3 | 해요체 | -해요, -아요 | 일반 대화, 비즈니스 |
| 4 | 하십시오체 | -습니다, -십시오 | 공식 자리, 격식 |

### 2.2 RBAC 매핑

경어 수준을 시스템 권한(Capability)으로 매핑한다:

```
하십시오체 → CAP_LEVEL 4 → 0b1000 → admin
해요체     → CAP_LEVEL 3 → 0b0100 → standard-user
해체       → CAP_LEVEL 2 → 0b0010 → peer
해라체     → CAP_LEVEL 1 → 0b0001 → system
```

### 2.3 CAP_REQUIRE 옵코드

모든 컴파일된 바이트코드 앞에 `CAP_REQUIRE` 옵코드가 삽입된다.
VM은 명령어를 실행하기 전에 현재 권한 수준을 검사한다.

```
; 소스: "레지스터 영에 42 대입하십시오"  (하십시오체)
; 컴파일:
[0000] CAP_REQUIRE 4     ; admin 권한 필요
[0001] LOAD_IMM   0, 42  ; R0 ← 42
```

해라체 수준(CAP 1)에서 하십시오체 문장을 실행하면 `CapabilityError`가 발생한다.
이는 문법 오류가 아니라 **권한 오류**다 — 한국어의 경어 체계를 RBAC으로 모델링한 핵심 결과다.

### 2.4 경어 일관성 검사

하나의 프로그램 내에서 경어 수준이 혼용되면 경고가 발생한다.
이는 실제 한국어 사용에서 경어 체계를 일관되게 유지하는 원칙과 같다.

```
경어 불일치: 문장 2 "계산을 해요..." (해요체) ≠ 문장 1 (하십시오체)
```

## 3. SOV→CPS 설계

### 3.1 SOV 어순

한국어는 주어-목적어-동사(SOV) 어순을 따른다:
동사가 항상 문장의 마지막에 온다.

```
[주어]       [목적어]     [동사]
레지스터 영에  42        대입하세요
```

### 3.2 Continuation-Passing Style과의 부합

CPS에서 함수는 결과를 반환하지 않고, continuation(다음에 할 일)에 전달한다:

```
; 명령형
result = add(3, 5)
print(result)

; CPS
add(3, 5, λresult. print(result))
```

SOV 구조가 CPS와 부합하는 이유:

1. **동사(명령)가 마지막** — CPS에서 continuation이 마지막 인자
2. **인자가 먼저 평가** — SOV에서 주어/목적어가 먼저 나옴
3. **연속적 결합** — 한국어 문장을 연결하면 자연스럽게 연속(continuation)이 됨

```
레지스터 영에 42 대입하세요
→ CPS: store(R0, 42, cont)

레지스터 일에 58 대입하세요
→ CPS: store(R1, 58, cont')

레지스터 영 더하기 레지스터 일
→ CPS: add(R0, R1, cont'')
```

### 3.3 컴파일 과정

```
소스: "레지스터 영에 42 대입하세요"
  │
  ├── SOV 파싱:
  │   arguments = ["레지스터", "영에", "42"]
  │   verb = "대입하세요"
  │
  ├── 경어 감지:
  │   "대입하세요" → 하십시오체 → CAP 4
  │
  └── 바이트코드 생성:
      CAP_REQUIRE 4
      LOAD_IMM 0, 42
```

## 4. 조사를 범위 연산자로

### 4.1 조사의 역할

한국어의 조사(후치사)는 명사에 붙어 문법적 역할을 결정한다.
이를 프로그래밍의 범위(scope) 연산자로 사용한다.

| 조사 | 문법 역할 | 범위 연산자 역할 | 예시 |
|------|-----------|-----------------|------|
| 은/는 | 주제 | 스코프 경계 | X는 ... (X 스코프) |
| 이/가 | 주어 | 액터 | X이 ... (X가 실행) |
| 을/를 | 목적어 | 타겟 | X를 ... (X를 조작) |
| 에게 | 방향 | 목적지 | X에게 ... (X에게 전송) |
| 의 | 소유 | 소유권 | X의 Y (X의 속성 Y) |

### 4.2 범위 예시

```
함장은 상황을 부함장에게 보고합니다
│     │     │
│     │     └── 목적지: 부함장에게 (SEND destination)
│     └── 타겟: 상황을 (SEND payload)
└── 스코프: 함장은 (scope boundary)
```

### 4.3 설계 의의

조사를 범위 연산자로 사용하면:
- **영어식 전치사**의 모호성을 해결
- **명사의 문법 역할**이 명시적
- **스코프 경계**가 문장 구조에 자연스럽게 내장

## 5. 활용을 함수 합성으로

### 5.1 동사 활용

한국어 동사는 어간에 여러 어미가 결합한다(교착어). 이를 함수 합성으로 모델링한다:

```
대입(어간) + 하(접미어) + 시(존칭) + 오(종결) = 대입하시오
store    ○ compose  ○ elevate  ○ finalize = elevated_finalized_store
```

어미가 쌓일수록 함수가 합성되어 더 복잡한 의미를 표현한다.

### 5.2 합성 규칙

| 어미 | 함수 효과 | 예시 |
|------|-----------|------|
| -하다 | 기본 실행 | 대입하다 → store() |
| -해요 | 사용자 권한 실행 | 대입해요 → store()@CAP3 |
| -하십시오 | 관리자 권한 실행 | 대입하십시오 → store()@CAP4 |
| -되다 | 수동/상태 전환 | 저장되다 → store_passive() |

## 6. 한자어를 타입명으로

한국어의 한자어(漢字語)는 학술·기술 용어로 널리 사용된다.
이를 FLUX의 공유 추상 타입 이름으로 사용한다:

```python
# 타입 정의 (math.ese)
정수 = "integer"      # 整數
실수 = "float"        # 實數
행렬 = "matrix"       # 行列
벡터 = "vector"       # 向量
함수 = "function"     # 函數
수열 = "sequence"     # 數列
집합 = "set"          # 集合
```

한자어를 타입명으로 사용하면:
- **국제적 공유 가능** — 한자 문화권(한국, 중국, 일본) 간 공유
- **의미의 명확성** — 한자의 의미가 타입의 의미를 직접 전달
- **한국어 친화적** — 영어 타입명 대신 한국어 타입명 사용

## 7. 바이트코드 명령어

### 7.1 산술 명령어

| 옵코드 | 이름 | 설명 |
|--------|------|------|
| 0x01 | ADD | 덧셈: ADD dst, a, b |
| 0x02 | SUB | 뺄셈: SUB dst, a, b |
| 0x03 | MUL | 곱셈: MUL dst, a, b |
| 0x04 | DIV | 나눗셈: DIV dst, a, b |
| 0x05 | MOD | 나머지: MOD dst, a, b |
| 0x06 | POW | 거듭제곱: POW dst, a, b |

### 7.2 권한 명령어

| 옵코드 | 이름 | 설명 |
|--------|------|------|
| 0x30 | CAP_REQUIRE | 권한 검사: CAP_REQUIRE level |
| 0x31 | CAP_SET | 권한 설정: CAP_SET level |

### 7.3 통신 명령어

| 옵코드 | 이름 | 설명 |
|--------|------|------|
| 0x40 | SEND | 개별 전송: SEND agent, message |
| 0x41 | BROADCAST | 전체 방송: BROADCAST message |
